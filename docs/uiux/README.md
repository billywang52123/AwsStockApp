# Handoff: 每日抽籤(御神籤式運勢籤)

## Overview
把原本「每天抽的陪伴卡」改成**日式搖籤(御神籤)**體驗:使用者每天搖一次籤筒,抽出一支「安心籤」,系統依其持股與當日市場,生成一張**籤詩結果頁**。

籤等分六級:**大吉 · 吉 · 小吉 · 小凶 · 凶 · 大凶**。每張籤固定三個欄位:
1. **持股與狀態** — 逐檔持股,每檔對應一個籤等
2. **說明** — 可能發生的事 + 是否適合進場
3. **注意事項** — 最近發生的事、需要注意的點

開籤瞬間會依「綜合籤等」出現氛圍特效:**偏吉 → 金光**、**偏凶 → 黑煙**,強度隨吉凶等級遞增;結果頁維持該氛圍(金光/黑煙自籤紙內緩緩漂出),大凶另有**滴黑雨**、大吉有**滴金雨**。

## About the Design Files
本包內的檔案是用 **HTML 製作的設計參考稿**(呈現預期外觀與動態的原型),**不是可直接複製上線的產品程式碼**。任務是**在目標 App 既有環境中重建這些設計**——此專案定位為 iOS 原生 App(SwiftUI,Light Mode);籤詩結果頁是深色沉浸畫面。請沿用 codebase 既有的元件、色彩與動畫模式重製;若尚無環境,再選擇最合適框架實作。

`股感安心卡 設計畫布.dc.html` 是一個設計畫布,內含多輪迭代;**本次交付的最終方向是第十二輪 `14a–14c–14d`**(老籤紙 · 發光書法 · 煙/光自籤內漂出)。第十輪(12a–12d)與第十一輪(13a–13d)是早期探索,僅供理解演進脈絡,**不需實作**。

## Fidelity
**High-fidelity(hifi)**。色彩、字體、間距、動畫時序皆為最終規格,請像素級重建。字體以 Google Fonts 為原型(見下),iOS 上請替換為對應的書法/明體字型檔並保持視覺一致。

## Interactions & Behavior
- **每日一籤**:每天限抽一次,收盤後 14:30 更新可再抽;已抽當日顯示當日結果。
- **抽籤流程**:入口(14a)→ 點「搖籤問卜」觸發 medium haptic + 搖籤過場(14b,~2–2.4s)→ 籤支彈出瞬間依綜合籤等播放金光/黑煙特效 → 淡入 + 上移展開籤詩結果頁(14c/14d)。
- **綜合籤等 → 特效強度**(六級階梯):
  - 大吉:全螢幕金光爆閃 + 旋轉光苒(閃瞎眼)
  - 吉:溫暖金色光暈罩頂,穩定發亮
  - 小吉:一小圈微光,溫和不刺眼
  - 小凶:一縷輕煙飄過即散
  - 凶:兩三縷黑煙升起、畫面轉暗
  - 大凶:濃煙滿版翻騰、畫面最暗 + 滴黑雨
- **情緒安全**:凶/大凶採低飽和灰黑、不突然閃爍,配安撫文案(如「先不做決定,也是一種決定」)與呼吸節奏;避免嚇到使用者。
- **綜合籤等演算**:由各持股狀態加權得出(演算法由後端/資料層決定,前端只呈現結果)。

## State Management
- `todayDrawn: bool` — 今日是否已抽(控制入口 vs 結果頁)。
- `drawState: 'idle' | 'shaking' | 'revealing' | 'result'` — 抽籤過場狀態機。
- `overallRank: enum(大吉…大凶)` — 綜合籤等,決定特效與配色主題。
- `holdings: [{ symbol, name, note, rank }]` — 逐檔持股與各自籤等。
- `explanation: { text, entryVerdict, entryNote }` — 說明欄位。
- `cautions: [string]` — 注意事項項目。
- `nextDrawAt` — 下次可抽時間(收盤後 14:30)。
- 資料來源:後端依持股 + 當日市場生成(含 AI 文案);前端負責過場、特效強度映射與呈現。
