單位 pt。基準 iPhone 16 Pro 402 × 874。頁面內距:上 78 / 左右 24 / 下 40(含底部固定免責帶)。
設計稿:`股感安心卡 設計畫布.dc.html` 第十四輪段落(16a–16e)。

## 核心概念

四件事串成一條資料鏈:

1. **問卷分型**(16a):5 題單選,把使用者分到四型之一,並落到四個維度分數(0–100)。
2. **風格顯示**(16b):把分型 + 四維度光譜呈現給使用者,並預覽「AI 之後會怎麼跟你說話」。
3. **風格 → Prompt**(16c):分型與維度、加上持股習慣標籤,注入 GPT system prompt。**風格只改語氣與重點排序,不改事實與數據。**
4. **習慣與轉變**(16d / 16e):從持股與更新紀錄「推算」投資習慣(免填寫);使用者更新持股後重算風格,並顯示「原風格 → 新風格」的轉變。

**四型定義**(內部 enum `InvestStyle`):

| enum | 顯示名 | 主色 | 一句話 |
|---|---|---|---|
| `steadyKeeper` | 穩健守成型 | `accent/primary #7B7FD4` | 看懂再行動,波動時先觀察 |
| `dividendCalm` | 存股安心型 | `updown/downText #6E9A7F` | 重配息與本業,少看短期股價 |
| `swingTrier` | 波段嘗試型 | `warn/amberNumber #C89A5A` | 願意抓事件與波段,仍謹慎 |
| `aggressiveRunner` | 積極衝刺型 | `styleProfile/aggressive #B0759A` | 資訊密度高、直接看數據 |

**四維度**(內部 `StyleAxis`,各 0–100,用光譜條呈現):
`risk` 風險承受 · `horizon` 持有期間 · `basis` 決策依據(短期價格 ↔ 公司本業)· `reaction` 波動反應(易慌 ↔ 冷靜)。

## 16c · 風格 → GPT Prompt 對應(開發交付,非 UI 畫面)

這是給後端/AI 串接的規格卡,設計稿裡以 402 寬白卡展示。

- **組裝時機**:每次要生成個股解讀 / 抽卡文案 / 每日首頁摘要時,即時把使用者當前 `styleProfile`(分型 + 四維度 + 習慣標籤)注入 system prompt。
- **system prompt 模板**(變數以 `{}` 包住,實際以字串插值):

  ```
  你是安撫優先的台股解讀助手。
  使用者風格:{style_name}
  維度:風險承受 {risk}｜持有期間 {horizon}
  　　　決策依據 {basis}｜波動反應 {reaction}
  持股習慣:{habit_tags}
  語氣規則:{tone_rules}
  禁用詞:買進、賣出、加碼、停損、目標價
  任務:用 3 句話解釋今日 {ticker} 波動原因,
  先講事實、再講與使用者的關聯、最後安撫。
  ```

- **四型 `tone_rules` 對照**:

  | 分型 | tone_rules(注入字串) |
  |---|---|
  | 穩健守成型 | 先安撫再解釋,強調長期持有理由未變 |
  | 存股安心型 | 重點放配息與本業,少提短期股價 |
  | 波段嘗試型 | 補充事件時間點與波動區間,多一句風險提醒 |
  | 積極衝刺型 | 資訊密度高、直接講數據,仍不給操作指令 |

- **鐵則**:風格只影響「語氣 + 重點排序 + 資訊密度」,**事實、數字、資料日期一律不變**;四型看到的同一事件,數據必須完全相同。
- **審核**:輸出過濾禁用詞(命中則要求模型重寫);全程不得出現目標價、預測漲跌、買賣建議。
- **重算生效點**:持股更新觸發風格重算後,新的 `{style_name}` 與 `{habit_tags}` 於**下一次生成**生效(不回溯改寫已產出的歷史卡)。

1. 漲跌色遵守台股慣例(漲紅跌綠、降飽和);**風格維度箭頭 ≠ 漲跌,不套紅綠,一律用中性 amber `#C89A5A` 表方向**。

3. 語氣安撫優先:全線不出現「買/賣/加碼」;風格分類不評價使用者(「沒有好壞」);每頁尾掛免責帶。
