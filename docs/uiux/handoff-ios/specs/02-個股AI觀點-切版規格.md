# 02 · 個股 AI 觀點 切版規格(8d / 8e / 17a)

單位 pt。8d 頁內距:上 88 / 左右 24 / 下 40;8e / 17a(詳情頁)上 80。

> **第十五輪整併(以 17a 為準,施作優先)**
> 過去一檔股票有三個入口、兩種詳情版型:①持股列表點入的「白話狀態翻譯器」②庫存分析內文點入的「8e 個股觀點詳情」③個股觀點 tab(11f)點入的同款內容。三者說法重疊、使用者困惑。
> 整併結論:**一檔股票只有一頁詳情 `StockDetailView`(17a),三個入口皆深連結進入同一頁**。
> - 白話狀態翻譯器 → 併入 17a 上半部「今天怎麼了?」區塊,不再是獨立頁。
> - **8e 個股觀點詳情退役** → 內容併入 17a 下半部「AI 怎麼看接下來?」;庫存分析內文不再有第二種詳情版型(仍保留下方 8e 規格供對照與資料欄位參考,實作以 17a 為準)。
> - 個股觀點 tab(11f)保留但**降級為列表**,價值是「一次掃過全部持股+觀察」,點任一列進 17a,自身不放詳情內容。
> 詳見本檔末段「## 17a · 統一個股詳情頁」。

### 建議元件命名
- `StockDetailView` — 整頁容器(取代原 `StockOutlookDetailView`)
- `TodayStatusCard` — 上半部「今天怎麼了?」白卡(內含既有 `ExplanationBlock` 新手翻譯盒)
- `AIOutlookCard` — 下半部 AI 綜合觀點卡(含 `SentimentMeter` + 內嵌 `SignalRow` ×N)
- `SignalRow` — 卡內單列訊號(取代原獨立 `NewsSignalCard`;若他處仍需卡片式可保留 `NewsSignalCard`)

### 遷移註記
- 原 `StockOutlookDetailView`(8e)route 併入 `/stock/:id`,舊深連結 301 到新 route。
- 持股列表列點擊行為由「開翻譯器頁」改為「開 `StockDetailView`」。
- 個股觀點 tab(11f)列點擊行為由「開 8e」改為「開 `StockDetailView`」;tab 本身只保留列表 + 評分 chip。
