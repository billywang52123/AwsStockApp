# 08 · 語音輸入加入持股 切版規格(19a / 19b / 19c)

單位 pt。基準 iPhone 16 Pro 402 × 874。頁面內距:上 88 / 左右 24 / 下 40。
設計稿:`股感安心卡 設計畫布.dc.html` 第十七輪段落(19a–19d)。對應架構見 `StockMood 架構圖.dc.html`「5 · Voice Holdings Input」。

## 核心 UX 原則

1. **裝置端轉文字,錄音不上傳**:語音只在 iPhone 上轉成文字(`SFSpeechRecognizer`,`requiresOnDeviceRecognition = true`),後端只收得到文字,不經手錄音檔;全程需在畫面上明講這件事,安撫隱私疑慮。
2. **文字才是唯一真相來源**:轉出的文字直接送 AI 解析,解析結果永遠可看見原句、可修改、可重說、可退回手動輸入,不強制用戶接受辨識結果。
3. **口語數字明確換算並標註**:「兩張」「九百八」等口語量詞/數字,換算成股數與成本價後,一律在卡片上用小字註記換算依據,不能悄悄轉換。
4. **成本價是選填**:句子沒提到成本,不擋加入流程,改用「補上成本(選填)」引導,不用錯誤提示。
5. **失敗不用錯誤紅**:辨識或解析失敗,一律安撫語氣文案 + 可重試 / 可改手動,沿用 7 系列的原則。

沿用 7c 骨架(持股確認列表),差異如下。

- 大標「聽懂了,是這 N 檔嗎?」28 Heavy(N 動態);副標 14 `ink/tertiary`:「確認一下,認錯了可以直接改」
- **原句引用盒**(距副標 18):`#F1F1FB` R16,內距 13/16,13pt `#4A4770` 行高 1.7,麥克風 icon 15pt 置左上;內容「你說:「{逐字稿全文}」」,逐字不省略
- **持股列**(距引用盒 16,列間 12,R18 內距 16/18 白底,規格同 7c):
  - 一般列:頭像圈 44 + 名稱/代號 + 右對齊股數(16 Bold tabular)+ 鉛筆 icon 14pt
  - **含成本列**:股數下方多一行 12pt `ink/tertiary` tabular「成本 {price} 元/股」;卡底加一條換算註記盒(`bg/canvas` R10,內距 8/12,12pt `ink/tertiary`),文案例:「『兩張』= 2,000 股 ·『九百八』= 成本 980 元」
  - **未提成本列**:卡底同位置改為 inline 提示「沒聽到成本價」+ 右側「補上成本(選填)」`accent/primary` 12 Bold 連結,點擊展開成本輸入(不擋加入)

- **雙選項列**(距列表 14,置中 gap 18,兩者點擊區 ≥44 高):
  - 「再說一次」14 Bold `accent/primary` + 麥克風 icon 15pt — 回到 19b 重新錄音
  - 「有漏的?手動補上」14 Bold `accent/primary` — 進手動輸入,結果保留
- **CTA「加入這 N 檔持股」**:高 54 R18,規格同 7c;底部沿用同款免責文案(11pt `ink/faint`:「解析結果僅供輸入輔助,加入前請確認與實際持股一致」)

## 解析欄位與換算規則(工程實作)

| 項目 | 規則 |
|---|---|
| 語音轉文字 | `SFSpeechRecognizer`,`requiresOnDeviceRecognition = true`,語系 zh-TW;partial result 即時回填 19b 逐字稿卡 |
| 傳輸 | 轉出的純文字 `POST /holdings/parse`(HTTPS + JWT),**不上傳音檔** |
| AI 解析 | Amazon Bedrock(Claude)NLU,輸出結構化陣列:每筆 `{股票代號, 股數, 成本價(可為 null), 動作:"加入"}` |
| 量詞換算 | 「張」固定 = 1,000 股;口語數字(如「九百八」→ 980、「兩張」→ 2,000 股)換算後一律在 19c 卡片標註換算依據 |
| 成本價缺失 | 該筆 `成本價 = null`,19c 顯示「沒聽到成本價」引導補填,不阻擋加入 |
| 靜音自動結束 | 偵測 3 秒無新語音輸入 → 自動觸發「說完了」,等同手動點擊,light haptic |
| 辨識/解析失敗 | 不顯示錯誤紅;文案「沒聽清楚,再說一次試試」+「再說一次」按鈕,或導向手動輸入 |
| 麥克風權限被拒 | 19a CTA 點擊後偵測權限,被拒則導向截圖/手動入口,附「去設定開啟」連結,不強制彈系統對話框以外的攔阻畫面 |

## 動畫

| 對象 | 規格 |
|---|---|
| 19a→19b 進場 | sheet 由下滑入,spring 0.45s + medium haptic |
| 麥克風脈衝環 | scale 1→2、opacity 0.5→0,`micPulse` 1.8s ease-out,雙環錯開 0.6s |
| 音量波形 | `vbar` scaleY 0.25↔1,0.9s ease-in-out,7 條依序錯開 0.12s |
| 逐字稿游標 | 寬 2,blink 1s |
| 解析中(19b→19c 過場) | 沿用 7b 三步驟進度(聽寫完成 → AI 解析 → 對應股名股數),全部完成停 0.4s 再 push |
| 19c 持股列進場 | fade + 上移 12pt,stagger 60ms/列 |
| 低信心 / 缺成本提示 | 列進場後延遲 0.6s 淡入(安撫原則:不搖晃、不用紅色) |
| CTA | 按下 scale 0.97;成功打勾 + medium haptic 後返回持股頁 |

三鐵則沿用 00-通用規格:降飽和紅綠、`monospacedDigit`、Dynamic Type、安撫語氣。
