# 04 · 持股異動與多券商合併 切版規格(9a / 9b / 9c / 9d / 9e)

單位 pt。基準 iPhone 16 Pro 402 × 874。頁面內距:上 88 / 左右 24 / 下 40。
設計稿:`股感安心卡 設計畫布.dc.html` 第七輪段落(9a–9f)。

## 核心 UX 原則

1. **先問意圖,再問數字**:所有持股異動一律先經 9a 意圖選擇(加碼 / 賣出 / 覆蓋),不給用戶直接改股數欄位,避免「取代還是攤平」語意不明。
2. **即時預覽**:輸入股數/價格時,結果卡(股數、均價、已實現損益)即時重算,用戶按下確認前就看得到後果。
3. **智慧預設**:匯入合併每檔給預設選項(同券商→取代、不同券商→分帳加總),用戶不動預設也能一鍵完成。
4. 破壞性動作(取代全部、刪除分帳)一律二次確認、可還原。

## 合併與均價計算規則(工程實作)

| 情境 | 規則 |
|---|---|
| 加碼買進 | 新均價 = (舊股數×舊均價 + 新股數×買價) ÷ 總股數,四捨五入至 0.1 |
| 加碼未填買價 | 只加股數、均價不變;持股掛 `avgPriceIncomplete` 標記,列表提示補填 |
| 賣出 | 股數減少、均價不變;已實現損益 = (賣價 − 均價) × 股數 |
| 全部賣出 | 持股移至「已出場」(soft delete,可還原);觸發 AI 覆盤卡 |
| 覆蓋為最新庫存 | 總股數取代、均價保留;確認彈窗顯示差異股數 |
| 匯入・同券商同檔 | 視為最新快照 → 預設「取代該券商分帳」 |
| 匯入・不同券商同檔 | 建立新分帳 → 預設「分帳加總」;總覽均價 = 各分帳加權 |
| 匯入無均價 | 分帳均價 null,不計入加權(分母扣除該分帳市值),提示補填 |
| 取代全部 | 刪除該檔其他分帳;destructive confirm + undo |
| 每筆異動 | 寫入 `HoldingActivity` log(型別/股數/價格/來源/時間),9e 可回看、左滑刪除並回算 |

## 動畫

| 對象 | 規格 |
|---|---|
| 9a sheet 進場 | 系統 sheet spring;選項列 fade + up 10pt · stagger 50ms |
| 意圖列點擊 | scale .97 + light haptic → push 對應輸入頁 |
| 9b/9c 預覽重算 | 數字 count-up 滾動 0.4s easeOut;舊值刪除線 fade in 0.2s;每次輸入 debounce 150ms |
| 均價箭頭 | 新值變動時箭頭 x 位移 3pt 往返 0.3s(提示方向) |
| 已實現損益盒 | 首次出現 fade + up 8pt;正負切換時色彩 crossfade 0.25s |
| 9d 選擇切換 | segment 描邊/底色 0.2s easeOut;算式盒內容 crossfade 0.25s(加總式 ↔ 取代式)|
| 9d CTA 成功 | 勾 path draw 0.3s + medium haptic → 返回持股列表,受影響列高亮 #EEEEFA 淡出 1.2s |
| 9e 占比 bar | 進場由 0 展開至實際占比 0.6s easeOut,stagger 80ms |
| Undo toast | 底部滑入,停留 5s;字 13,含「還原」按鈕(≥44 高)|
