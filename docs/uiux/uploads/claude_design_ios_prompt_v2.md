# Claude Design Prompt：iOS 原生 App UI / UX 設計

> 用途：貼給 Claude Design / UI 設計 Agent。  
> 方向：只設計 iOS 原生 App，不考慮 Web、不考慮 Android、不考慮跨平台。  
> 目標：產出完整 iOS App 畫面、Design System、互動設計與文案。

---

## 角色設定

你是一位資深 **iOS App UI/UX Designer**。

請幫我設計一款 iOS 原生 App。

產品名稱暫定：

```text
股感安心卡
```

這是一款給投資新手使用的情緒陪伴型股票 App。

使用者通常已經持有 1~5 檔股票，但看不懂線圖、新聞和市場原因，看到股票漲跌會焦慮。

App 每天幫使用者整理：

1. 今日焦慮分數
2. 我的持股為什麼漲跌
3. 個人持股 vs 大盤
4. 每日抽卡
5. 白話市場解釋

請以 **iOS 原生體驗** 設計。

不考慮：

- Web
- Android
- React Native
- Flutter
- 桌機版
- 股票下單功能

## 一、產品背景

目標使用者是投資新手，但不是完全沒買過股票的人。

他們通常已經持有約 1~5 檔股票，例如：

- 台積電
- 鴻海
- 聯發科
- 金融股
- ETF

但他們：

- 看不懂線圖
- 看不懂財經新聞
- 不知道新聞和股價有什麼關係
- 看到股票跌就焦慮
- 看到股票漲也會擔心自己是不是做錯決定
- 不知道是自己選錯股票，還是整個市場都在跌
- 不需要複雜分析，只需要有人用白話幫他解釋

核心價值：

```text
每天用白話告訴你，今天你的股票為什麼漲跌，以及你需不需要那麼焦慮。
```

## 二、產品定位

這不是券商 App。

這不是下單 App。

這不是專業投資分析工具。

請設計成：

- 投資新手的每日情緒陪伴 App
- 白話市場翻譯器
- 每日持股情緒整理工具
- 有抽卡儀式感的股票陪伴工具

要像：

- 每日陪伴工具
- 情緒安撫工具
- 簡單理財習慣 App
- 有溫度的個人小助理

不要像：

- 專業券商 App
- 股票下單 App
- 複雜技術分析工具
- 冷冰冰的金融報表
- 密密麻麻的股價列表

請使用：

- iOS native feel
- SwiftUI friendly
- 溫暖
- 簡單
- 新手友善
- 卡片式 UI
- 有每日儀式感

- 可信任但不冰冷
- 清楚但不專業壓迫

視覺關鍵字：

```text
soft finance
emotional companion
beginner friendly
daily ritual
card-based UI
calm but not boring
warm and trustworthy
iOS native
```

- 米白
- 淺藍

- 淡橘

- 大量紅綠 K 線

### 3. 使用者情境選擇

用途：

- 讓使用者覺得 App 懂他的焦慮

標題：

```text
你最常遇到哪種情況？
```

選項卡片：

```text
我買了股票，但不知道該不該繼續放著
我看不懂線圖和新聞
股票一跌，我就開始焦慮
我只是想每天快速知道發生什麼事
```

CTA：

```text
下一步
```

設計建議：

- 使用 iOS 卡片列表
- 選中狀態可以有柔和描邊
- 可使用 SF Symbols 表示情緒或場景

---

### 4. 持股輸入

用途：

- 讓使用者輸入 1~5 檔股票

標題：

```text
先加入你關心的股票
```

說明：

```text
不用輸入金額也可以先開始，我們會先幫你看懂今天的波動。
```

輸入欄位：

- 股票代號或名稱，例如 2330、台積電
- 可選：持有成本
- 可選：持有股數

已加入股票卡片：

- 股票名稱
- 股票代號
- 移除按鈕

CTA：

```text
完成，查看今日狀態
```

設計重點：

- 輸入流程要很輕
- 不要像金融開戶流程
- 讓使用者覺得「可以先隨便試試」

---

### 5. AI / 規則推薦股票

用途：

- 降低使用者輸入門檻
- 增加互動感

標題：

```text
你可能也關心這些股票
```

說明：

```text
根據你剛剛加入的股票，我們猜你可能也想追蹤這些。
```

推薦卡片內容：

- 股票名稱
- 股票代號
- 產業標籤
- 加入按鈕
- 略過按鈕

範例：

使用者輸入台積電後，推薦：

```text
聯發科
鴻海
日月光
廣達
緯創
```

設計建議：

- 推薦卡片橫向滑動
- 卡片可以有淡淡光澤或柔和陰影
- 加入後給一個小型成功回饋

---

### 6. 今日焦慮分數首頁

用途：

- App 主首頁
- 使用者每天回訪的核心畫面

標題：

```text
今天你的持股情緒
```

主要區塊：

- 今日焦慮分數，例如 72 / 100
- 情緒標籤：「有點緊張」
- 一句白話摘要

摘要範例：

```text
今天你的持股波動偏大，主要受到科技股拉回影響。
```

次要區塊：

- 我的持股今日表現
- 大盤今日表現
- 個人持股 vs 大盤比較
- CTA：「查看今天原因」
- CTA：「抽今日持股卡」

設計重點：

- 焦慮分數是主視覺
- 可以用圓形儀表、情緒溫度計或柔和進度環
- 不要讓分數看起來像嚴重警報
- 文字要有安撫感

---

### 7. 每日抽卡

用途：

- 增加每日回訪感
- 建立互動記憶點

標題：

```text
抽一張今日持股卡
```

畫面中央：

- 一張尚未翻開的卡片

互動狀態：

1. 點擊卡片
2. 卡片翻面
3. 出現柔和光效
4. 觸覺回饋
5. 顯示今日卡片結果

卡片類型：

```text
冷靜觀察卡
大盤影響卡
小心震盪卡
個股事件卡
信心恢復卡
```

卡片結果內容：

- 卡片名稱
- 一句安撫文案
- 一句市場原因
- CTA：「查看詳細原因」

範例文案：

```text
今天不是只有你的持股波動，整體市場也偏弱。先看懂原因，比急著做決定更重要。
```

設計建議：

- 翻牌動畫要自然
- 光效不要太遊戲化
- 要有每日儀式感
- 適合 SwiftUI animation 實作

---

### 8. 今日原因解釋

用途：

- 用白話解釋今天使用者持股為什麼漲跌

標題：

```text
今天為什麼會這樣？
```

區塊 1：今日一句話

```text
今天你的持股下跌，主要是受到整體科技股拉回影響。
```

區塊 2：白話解釋

```text
這不一定代表你選錯股票，而是今天市場對科技股比較保守，相關股票一起受到影響。
```

區塊 3：我的持股影響排行

```text
台積電：影響最大
鴻海：影響中等
中信金：影響較低
```

區塊 4：和大盤相比

```text
我的持股：-1.8%
大盤：-0.9%
今天你的持股比大盤弱一些，主要是科技股比重較高。
```

區塊 5：免責聲明

```text
內容僅供資訊參考，不構成投資建議。
```

### 9. 我的持股列表

用途：

- 讓使用者管理持股

標題：

```text
我的持股
```

股票卡片內容：

- 股票名稱
- 股票代號
- 今日漲跌百分比
- 對焦慮影響：高 / 中 / 低
- 狀態標籤，例如「大盤影響」、「個股波動」、「資料不足」
- 查看詳情
- 移除

底部 CTA：

```text
新增股票
```

### 10. 個股詳情

用途：

- 解釋單一股票今天的狀況

標題範例：

```text
台積電今天怎麼了？
```

區塊：

- 今日漲跌
- 與大盤比較
- 可能原因
- 新手翻譯
- 對焦慮分數的影響

範例文案：

```text
台積電今天下跌 1.2%，比大盤稍弱。
這比較像是科技股整體拉回，不一定是公司單獨出問題。
```

CTA：

```text
回到今日總結
```

### 12. Loading

用途：

- 資料整理中

- 計算焦慮分數中

文案：


### 13. Empty / Error

用途：

- 尚未輸入持股
- 資料不足
- 股票查不到

Empty 文案：

```text

先加入 1 檔你關心的股票，我們會幫你整理今天的波動原因。
```

Error 文案：

```text
今天這檔股票的資料還不完整
你可以稍後再試，或先查看其他持股。
```

CTA：

```text
新增股票
重新整理
回首頁
```
